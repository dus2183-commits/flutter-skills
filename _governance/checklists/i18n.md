# 国际化检查清单

> 用于 `flutter-review` 检查国际化合规。

---

## A. 硬编码字符串

- [ ] 没有任何中文字符串硬编码在 dart 文件中
  - `grep -rE "[\u4e00-\u9fa5]" lib/` 应只在 i18n 文件出现
- [ ] 没有英文 UI 文案硬编码 (除 debug print)
- [ ] AppBar title 用 `'xxx.title'.tr`
- [ ] Button text 用 `.tr`
- [ ] Tooltip / placeholder / hint 用 `.tr`
- [ ] Snackbar / Dialog 文案用 `.tr`

---

## B. Key 命名

- [ ] key 用 snake_case (`announce_list_title` 不是 `AnnounceListTitle`)
- [ ] key 按模块分组 (`announce.list_title`,模块名前缀)
- [ ] key 不重复 (一处定义,多处复用)
- [ ] 临时缺 key 用 `// i18n: TODO` 标记

---

## C. 文件组织

- [ ] 文件按语言分目录: `lib/app/locales/zh_cn/` / `en_us/`
- [ ] 每个模块一个 dart 文件: `announce.dart`
- [ ] 一个 `translations.dart` 主文件聚合
- [ ] 注册到 GetMaterialApp 的 `translations`

```dart
// lib/app/locales/translations.dart
class AppTranslations extends Translations {
  @override
  Map<String, Map<String, String>> get keys => {
    'zh_CN': zh_CN.translations,
    'en_US': en_US.translations,
  };
}
```

---

## D. 参数化文案

- [ ] 含变量的文案用 `.trParams`
- [ ] 不要字符串拼接 ("共 " + count + " 条")
- [ ] 用 `'common.total_count'.trParams({'count': '$count'})`
- [ ] 复数形式用 `.trPlural` (有的语言复数规则不同)

---

## E. 日期 / 时间 / 数字

- [ ] 日期格式化用 `intl` 包,不手写 `'$year-$month-$day'`
- [ ] 时区用 `DateTime.toLocal()` 或显式 timezone
- [ ] 数字格式化用 `NumberFormat` (千分位 / 货币)
- [ ] 时间戳显示 "刚刚 / N 分钟前 / 昨天" 用 i18n 文案

---

## F. RTL 布局

- [ ] 不用 `EdgeInsets.only(left: x)`,用 `EdgeInsetsDirectional.only(start: x)`
- [ ] 不用 `Alignment.centerLeft`,用 `AlignmentDirectional.centerStart`
- [ ] Icon 方向支持 RTL (有些 icon 要镜像)
- [ ] 测试: 临时切到阿拉伯语,检查布局

---

## G. 缺失 key 处理

- [ ] 缺失 key 显示 key 本身 (不显示空白)
- [ ] CI 检查脚本: 列出所有缺失 key
- [ ] 新加 key 时同步加所有语言

---

## H. 图片资源

- [ ] 图片中含文字的,按语言提供多版本 (`assets/image/zh_cn/` / `en_us/`)
- [ ] 或避免在图片上加文字,用 Text widget 叠加

---

## 严重度判定

| 违反 | 严重度 |
|---|---|
| 硬编码中文字符串 | ⚠️ 警告 (大量则升级 ❌) |
| key 不按规范命名 | ⚠️ 警告 |
| 字符串拼接而非 trParams | ⚠️ 警告 |
| 日期手写格式化 | ⚠️ 警告 |
| 缺失 key 不报错 | ⚠️ 警告 |
| 不支持 RTL 布局 | ⚠️ 警告 (除非业务不需要) |
